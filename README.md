E-Commerce Order Management Database System – Week 1

Project Overview

This project focuses on understanding the business requirements of an E-Commerce Order Management Database System. The purpose of Week 1 is to analyze the business scenario,
identify the system requirements, and prepare the necessary documentation before designing the database. The project aims to develop a centralized database that efficiently manages
customer information, product details, supplier records, orders, payments, shipments, and customer reviews.

Business Scenario

A rapidly growing e-commerce company requires a database system to manage its daily operations effectively. The proposed system will support customer registration, product management,
inventory tracking, order processing, payment management, shipment tracking, and customer reviews. By storing all business information in a centralized database, the organization can reduce data redundancy, improve data accuracy,
increase operational efficiency, and generate meaningful business reports.

Week 1 Objectives

During Week 1, the business problem was analyzed to understand the organization's requirements. The project involved identifying the major business processes,
stakeholders, functional requirements, non-functional requirements, project scope, assumptions, and constraints. These findings were documented in the form of a Requirement Analysis Report, 
Business Requirement Document (BRD), and Software Requirement Specification (SRS).

Core Entities

The project uses the predefined entities provided in the assignment. These include Customer, Category, Product, Supplier, Order, Order Details, Payment, 
Shipment, and Review. These entities will be used throughout the project for database design, ER modeling, normalization, SQL implementation, and report generation.

Deliverables

The Week 1 deliverables include the Requirement Analysis Report, Business Requirement Document, Stakeholder Analysis, Functional Requirements, 
Non-Functional Requirements, Project Scope, Assumptions and Constraints, Software Requirement Specification (SRS), and this README document. 
The outcome of Week 1 is a clear understanding of the business problem and a complete set of requirement documents that serve as the foundation for the database design 
and implementation in the upcoming weeks. This analysis ensures that the proposed database system meets the business needs and supports efficient e-commerce operations.
